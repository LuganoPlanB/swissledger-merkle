## Architecture

**Active Root Model**: only one Merkle root (`activeRoot`) lives on-chain. Updating it invalidates all previous proofs. Storage is O(1).

- `MerkleRootRegistry` ŌĆö the contract. Controls `activeRoot` and an owner-managed allowlist of root updaters.
- `StrkMerkleProof` ŌĆö library matching `@ericnordelo/strk-merkle-tree` semantics (sorted-pair Pedersen hashing).
- `StarknetPedersen` / `EllipticCurve` ŌĆö on-chain Pedersen hash implementation over the Starknet field prime.
- `BuildInfo` ŌĆö auto-generated contract embedding the npm package version (exposed via `version()`).

## SwissLedger Chain (ledger.swiss)

### Chain identity

| Property | Value |
|---|---|
| Chain ID | `110` (0x6e) |
| Explorer | `https://explorer.ledger.swiss` |
| RPC endpoint | `https://explorer.ledger.swiss/api/eth-rpc` |
| Block gas limit | ~20,000,000 |
| Gas price | Always 0 (permissioned, gas-free) |
| EVM level | **Istanbul** (pre-Shanghai, no `PUSH0`, no `MCOPY`) |
| Explorer API | Blockscout at `https://explorer.ledger.swiss/api` |

### EVM level: Istanbul

The chain runs an **Istanbul** EVM ŌĆö it does **not** support `PUSH0` (Shanghai EIP-3855)
or `MCOPY` (Cancun EIP-5656). When these opcodes are hit, the EVM treats them as
**INVALID** and consumes **all remaining gas** ŌĆö the tx fails with the gas-limit
consumed and no revert reason.

`foundry.toml` defaults to `evm_version = "istanbul"`, so `forge build` and
`forge test` produce compatible bytecode by default. The `via_ir` flag is still
required (the Pedersen arith libs hit "stack too deep" without it).

### RPC quirks

The Blockscout `/api/eth-rpc` endpoint requires every JSON-RPC request to
include the `"params"` field (even when empty) and returns non-standard error
responses. These are handled natively by `swissledger-cast` ŌĆö see
[FOUNDRY.md](./FOUNDRY.md).

### Deploying to ledger.swiss

Preconditions:
- Wallet registered on the chain (via the `FULL` registrar tx)
- Private key in `evm-private-key.txt` (generated by `scripts/keygen`)

```bash
# 1. Compile (istanbul is default in foundry.toml)
forge build

# 2. Deploy
forge create src/MerkleRootRegistry.sol:MerkleRootRegistry \
  --rpc-url https://explorer.ledger.swiss/api/eth-rpc \
  --private-key "0xŌĆ”" \
  --gas-limit 5000000 \
  --gas-price 0 \
  --legacy \
  --broadcast
```

**Important**:
- Always use `--legacy` (the chain rejects EIP-1559 type-2 transactions with
  "transaction type not supported").
- Always use `--gas-price 0` (the chain rejects non-zero gas price with
  "Gas price not 0").
- Gas limit ~5M is sufficient for `MerkleRootRegistry` (actual usage ~2M).
- The block gas limit is ~20M; the contract fits comfortably.
- Check transaction at `https://explorer.ledger.swiss/tx/<hash>`.
- If gas used equals the limit, the contract likely hit an unsupported opcode.
  With the istanbul EVM target this should not happen.

### Calling the deployed contract

Read-only calls work directly via the RPC:

```bash
# version()
curl -s -X POST "https://explorer.ledger.swiss/api/eth-rpc" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"eth_call","params":[{"to":"0xŌĆ”","data":"0x54fd4d50"},"latest"],"id":1}'

# owner()
curl -s -X POST "https://explorer.ledger.swiss/api/eth-rpc" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"eth_call","params":[{"to":"0xŌĆ”","data":"0x8da5cb5b"},"latest"],"id":1}'
```

State-changing calls (`setActiveRoot`, etc.) need signed transactions:

```bash
cast send "0xŌĆ”" "setActiveRoot(bytes32)" "0xŌĆ”ROOTŌĆ”" \
  --rpc-url https://explorer.ledger.swiss/api/eth-rpc \
  --private-key "0xŌĆ”" \
  --gas-limit 200000 \
  --gas-price 0 \
  --legacy
```

### Contract verification (Blockscout)

Verification via the API is currently unavailable ŌĆö the Blockscout instance
does not support the verification endpoint.

The flattened source is at:

```bash
forge flatten src/MerkleRootRegistry.sol > /tmp/MerkleRootRegistry_flattened.sol
```
